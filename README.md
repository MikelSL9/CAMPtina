# CAMPtina

Sistema web para la gestión integral del comedor de empresa desarrollado como proyecto final de DAW. Permite a los trabajadores seleccionar menús diarios (primero, segundo y postre), elegir turno de comida y gestionar sus reservas mediante autenticación segura. Incluye panel de administración para la creación de usuarios, platos, menú y turnos y modificación de todos los datos.

## Tecnologías utilizadas
- **Backend:** Spring Boot (Java), MySQL
- **Frontend:** React
- **Estado:** Context API
- **Autenticación:** JWT (JSON Web Tokens)

## Características principales
- Sistema de autenticación con login seguro
- Control de acceso basado en roles (usuario/administrador)
- Rutas privadas protegidas según permisos
- Selección de menú diario (primero, segundo, postre)
- Gestión de turnos de comida
- Panel de administración para CRUD de usuarios

## Mi rol en el proyecto
Responsable del **desarrollo frontend, autenticación e integración API**:
- **Sistema de autenticación y login**: Desarrollo completo del flujo de login en React.
- **Rutas privadas por roles**: Implementación de protección de rutas según tipo de usuario (acceso diferenciado entre usuario estándar y administrador).
- **Componentes React**: Desarrollo de componentes para gestionar turnos y platos.
- **Gestión de estado**: Implementación de Context API para estado global y manejo de sesiones
- **Integración API-Frontend**: Conexión completa con backend usando Axios.
- **Backend**: Gestión de clases Java relacionadas con usuarios y datos.
